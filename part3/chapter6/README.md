# 3.6 Github-flow

GitHub-flowについて、世の中には様々な解釈があり、多様な運用がされている。
本書で推奨したいのは、GitHub社の従業員が推奨している運用方法。

決定的な違いとしては、masterにmergeする前にプロダクションにdeployすること。
deployした後に実際にモニターリングして、問題がなければ、初めてmasterにmergeする。

詳細については、各セクションにて詳しく記載する。
